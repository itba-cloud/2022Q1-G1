MHS - Grupo 1

El objetivo es automatizar el Sistema de consultas y registro de pacientes con una solución Cloud a través del uso de una página web dinámica junto con un chatbot. La solución permitirá escalar on-demand, distribuir la carga de pacientes, tener una comunicación entre los distintos consultorios de forma sencilla, tener mayor velocidad y agilidad, y evitar el registro de consultas de forma manual.

Servicios Implementados

Se implementaron distintos servicios de AWS que estaban definidos en nuestra arquitectura serverless, siguiendo las buenas prácticas de Terraform y teniendo en cuenta las restricciones del Lab de Academy

Lambda

Se hicieron algunas lambdas con data mockeada: getMedicalRecords(), postMedicalRecord(), postAppointment()

Api Gateway

Se tiene un API Gateway para hacer los requests llamando a las lambdas

CloudFront

Levantamos cloudfront como CDN para distribuir el contenido

DynamoDB

Originamente teníamos Aurora Serverless pero debido a las restricciones de Academy no logramos levantar esa base, nos faltaban permisos. Entonces decimos levantar dos tablas de DynamoDB (appointments-table y users-table)

S3

Esta página está hosteada en un bucket de S3. El www hace un redirect y tenemos un dominio pago para tener HTTPS. (Academy no nos deja levantar Route53 ni ACM)

Cognito

El boton de SignUp hace un redirect a Cognito, te lleva a su página y te deja inscribirte con username, mail y password. Te manda un mail para que verifiques tu user y después te trae a la página de nuevo (para esto era necesario tenes HTTPS)

WAF

Definimos un Web-ACL con una rule de OWASP para vulnerabilidades y levantamos WAF como firewall

CloudWatch

Levantamos una alarma para monitorear el API-GW y un topic de SNS default como ejemplo para probar este servicio

Esperamos que haya gustado! Para el final vamos a incorporar más cosas para no tener toda la data mockeada
